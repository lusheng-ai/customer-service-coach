# 智能客服陪练系统

一个基于 **结构化业务流程 + 大模型协作** 思路实现的智能客服陪练系统。  
项目从真实业务场景设计出发，围绕 **场景库、分类器、状态机、用户模拟器、业务评分模块** 搭建完整链路，用于模拟客服多轮接待、约束用户回复范围、控制流程推进，并对客服表现进行可解释的业务评分。

这个项目不是一个“单纯让模型聊天”的 Demo，而是一次围绕 **LLM 在真实业务流程中如何做到更可控、更稳定、更可评估、更可复盘** 的工程实践。

## 项目背景

在很多客服、陪练、流程指导类场景里，真正的难点往往不是“让模型说得像人”，而是：

- 如何让对话始终围绕业务，不偏题、不编造
- 如何让多轮流程按标准路径推进，而不是靠上下文自由发挥
- 如何让系统知道当前应该问什么、做什么、何时结束
- 如何对客服表现做客观评分，而不是主观印象打分
- 如何在对话结束后给出可解释、可复盘的评分依据

因此，这个项目采用了“**先把业务结构化，再接入大模型**”的设计方式：  
先定义业务场景的流程边界，再让模型承担自然语言理解和生成的部分能力。

---

## 项目目标

本项目主要解决两类问题：

### 1）模拟真实业务用户，进行客服陪练

- 系统模拟访客用户发起咨询
- 用户回复始终围绕业务场景，不随意偏离
- 不凭空捏造业务事实
- 支持多轮问询、步骤反馈、兜底收口

### 2）对客服进行业务流程质检评分

- 判断客服是否按标准流程处理问题
- 检测是否存在漏问、跳步、重复问、错误收口等问题
- 输出每轮评分依据，支持训练和复盘
- 为后续扩展情绪价值评分、服务规范评分等维度预留空间

---

## 核心设计思路

这个项目的核心不是“端到端让一个大模型完成全部任务”，而是把系统拆成多个职责清晰的模块协作完成：

- **场景库**：定义业务场景的结构化知识
- **客服分类器**：把客服自然语言翻译成结构化行为事件
- **用户分类器**：把用户自然语言翻译成结构化反馈事件
- **状态机**：控制整个多轮流程推进
- **用户模拟器**：根据当前回复意图生成用户话术
- **业务评分模块**：根据当前流程前沿对客服行为进行评分

可以把整个系统理解为：

> 用结构化系统管流程，用大模型做语言理解和语言生成。

## 系统架构

### 模块关系概览

```text
场景库
  ↓
初始化对话上下文
  ↓
客服输入一句话
  ↓
客服分类器 → 解析为 AgentEvent
  ↓
业务评分模块 → 判断本轮是否命中流程
  ↓
状态机 → 更新当前阶段、槽位、流程前沿
  ↓
用户回复意图构造
  ↓
用户模拟器 → 生成访客回复
  ↓
用户分类器 → 解析为 UserEvent
  ↓
状态机 → 再次更新上下文
  ↓
判断是否结束，否则进入下一轮
```
